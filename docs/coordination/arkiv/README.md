# Coordination-arkiv

Lukkede pakke-artefakter — KUN læsbar reference, aldrig aktiv kilde.

**Hvad bor her (§4 bevarelses-disciplin — politik, ikke garanti):** pr.
afsluttet pakke bevares krav-dokket og plan-slutversionen
(`<pakke>-krav-og-data.md` og `<pakke>-plan.md`); ældre pakker kan mangle den
ene (kun plan eller kun krav overlevede). Derudover bevares filer som aktive
docs refererer — pr. 2026-06-12: `H020-flow-fejl.md` og
`mathias-afgoerelser-historik.md` (governance-checkets pointer-vagt håndhæver
at de ikke dør). Slut-rapporter bor i `rapport-historik/`, ikke her.

**Alt andet lever i git-history:** mellemversioner (V1..Vn), runde-/verdikt-
filer, recon, arbejdsmapper og leverancer fra afskaffede arbejdsformer slettes
fra arbejdstræet ved pakke-luk — `git log` er audit-sporet og mister intet.
Arkivet er ikke en voksende kirkegård (§4); ligegyldig fylde er en glid-kilde
for AI-aktørerne (Mathias-retning 2026-06-11).
