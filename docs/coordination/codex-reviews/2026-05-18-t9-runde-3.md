## Mathias' summary

Codex identificerede **2 KRITISK fund** der skulle fixes før admin-merge:

### KRITISK 1 — Manglende GRANT på 6 write-tabeller

> "RLS policies are added, but the required table privileges are still missing. Existing T9 migrations revoked all and granted only SELECT to authenticated. RLS policy does not itself grant SQL privileges, so SECURITY INVOKER write-RPCs still fail with table permission errors before the session-var policy can authorize them."

### KRITISK 2 — SELECT-policy blokerer dispatcher i pending_change_approve + pending_change_undo

> "The existing SELECT policy only allows requester or is_admin() to see pending rows. In the new functions, select \* ... for update happens before the change_type -> page_key -> has_permission(..., true) dispatcher. A non-admin user with can_edit on org_nodes, employee_placements, or client_placements will not be able to load the pending row to reach the dispatcher."

Plus 2 mindre:

- Stale kommentar i RPC 2.11 (DELETE-policy ikke i scope)
