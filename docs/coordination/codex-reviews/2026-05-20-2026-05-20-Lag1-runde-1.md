### Dialog-protokol — FLAG → LØS → STOP med max-cap

```
FLAG    — Marker rejses (af enten Code eller Codex)
LØS     — Dialog, MAX 3 LØS-iterationer per fund
          Iter 1: initial respons + modrespons
          Iter 2: refine
          Iter 3: sidste forsøg
          Iter 4 (auto-eskalation): tving STOP
STOP    — Hvis nogen rejser ESCALATE, ELLER iter > 3
```

**Iteration-counting mechanik (V5.1 fix HUL B):**

- Iter tælles **PER FUND PER RUNDE** — ikke globalt
- Reset ved ny runde (samme fund kan re-rejses i V2-runde uden iter-arv)
- Codex' runde-output inkluderer metadata pr. fund:

```
[FUND-1] KRITISK-SIKKERHEDSHUL: ...
  Iter: 1/3
  Status: AKTIV
```

- Hvis fund når iter 3 uden konvergens → auto-eskalation marker tilføjet i sluttekst:

```
AUTO-ESKALATION — Runde N
[FUND-X] iter 3 nået uden konvergens. Mathias-judgment påkrævet.
```

name: Post comment to Codex review queue

steps: - uses: actions/checkout@v4
with:

      - name: Locate tracker issue
        id: tracker
        env:
          GH_TOKEN: ${{ github.token }}
        run: |
          set -euo pipefail
          NUMBER=$(gh issue list --search "Codex review queue in:title" --state open --json number,title --jq '.[] | select(.title == "Codex review queue") | .number' | head -n1)
          if [ -z "$NUMBER" ]; then
            echo "Tracker issue 'Codex review queue' not found. Skipping (must be opened manually once)." >&2
            echo "skip=true" >> "$GITHUB_OUTPUT"
          else
            echo "skip=false" >> "$GITHUB_OUTPUT"
            echo "number=$NUMBER" >> "$GITHUB_OUTPUT"
          fi

      - name: Detect trigger type
        id: trigger
        if: steps.tracker.outputs.skip != 'true'
        env:
          EVENT: ${{ github.event_name }}
        run: |
          set -euo pipefail

          if [ "$EVENT" = "pull_request" ]; then
            # Pre-merge slut-rapport-trigger; paths-filteret garanterer seneste-rapport.md er ramt.
            echo "type=slut-rapport-pr" >> "$GITHUB_OUTPUT"
            exit 0

          # Push-event: brug git diff til at finde ændrede filer.
          changed=$(git diff --name-only HEAD~1 HEAD || true)
          echo "changed files:" >&2
          echo "$changed" >&2

          plan_feedback_changed=$(echo "$changed" | grep -E '^docs/coordination/plan-feedback/.+\.md$' | head -n1 || true)
          aktiv_plan_changed=$(echo "$changed" | grep -E '^docs/coordination/aktiv-plan\.md$' || true)
          seneste_rapport_changed=$(echo "$changed" | grep -E '^docs/coordination/seneste-rapport\.md$' || true)

          if [ -n "$plan_feedback_changed" ]; then
            echo "feedback_file=$plan_feedback_changed" >> "$GITHUB_OUTPUT"
            case "$plan_feedback_changed" in
              *-approved.md)
                echo "type=plan-approved" >> "$GITHUB_OUTPUT"
                ;;
              *-blokeret.md)
                echo "type=plan-blokeret" >> "$GITHUB_OUTPUT"
                ;;
              *-codex.md)
                echo "type=codex-feedback" >> "$GITHUB_OUTPUT"
                ;;
              *-code.md)
                echo "type=code-feedback" >> "$GITHUB_OUTPUT"
                ;;
              *)
                echo "type=plan-feedback-other" >> "$GITHUB_OUTPUT"
                ;;
            esac
          elif [ -n "$aktiv_plan_changed" ]; then
            echo "type=ny-plan-version" >> "$GITHUB_OUTPUT"
          elif [ -n "$seneste_rapport_changed" ]; then

            codex-feedback)

            code-feedback)
              HEADER="**Code-feedback leveret** — \`$FEEDBACK_FILE\`"
              ACTION="Code har svaret. Codex's tur — committ næste \`-codex.md\` eller \`-approved.md\` hvis konvergens nået."

            plan-approved)

            echo "- Run: <$RUN_URL>"
