# Claude Code-egenskaber — vurdering ift. Stork 2.0

Gennemgang 2026-06-12 af den officielle dokumentation (code.claude.com, 146 sider,
version-kontekst v2.1.157), fire community-artikler samt OpenAI's Codex-plugin-repo.

| Egenskab (hvad det er)                                                                                           | Fordel (hvad det gør)                                                                                                | Udbytte (hvad vi får ud af det ift. Stork 2.0)                                                                               |
| ---------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **Hooks** — scripts der kører automatisk ved faste livscyklus-punkter (Stop, SessionStart, TaskCompleted m.fl.)  | Udfører, blokerer eller validerer hver gang — garanteret, uafhængigt af om modellen husker det                       | KAEDE-STOP og git-sync (disciplin §13) håndhævet hårdt i stedet for prosa-disciplin der kan glemmes                          |
| **`/goal`** — kommando der sætter en målbar slutbetingelse for sessionen (v2.1.139+)                             | Arbejder videre tur efter tur, indtil en uafhængig evaluator bekræfter at betingelsen holder; virker også uovervåget | Done-betingelse pr. batch defineret FØR arbejdet; selftest RØD→GRØN bevist i output før batchen lukkes                       |
| **`.claude/rules/`** — mappe i repoet med regelfiler, der kan afgrænses til fil-stier                            | Loader reglerne automatisk når matchende filer læses, uden at fylde CLAUDE.md                                        | Konflikthierarki og LÅST-AUTORITATIV-status følger automatisk med strategi-/coordination-filerne; regelændringer via PR      |
| **Skills** — markdown-procedurer i `.claude/skills/` der bliver til `/kommandoer`                                | Udfører gentagne procedurer ens hver gang med ét kald; kan låses til kun eksplicit kald                              | Preflight, rolle-instruks og gate-tjek som faste kommandoer i stedet for prosa-instrukser                                    |
| **Codex-plugin** — officielt OpenAI-plugin der gør Codex tilgængelig inde i Claude Code                          | Kører review, adversarial review og opgave-delegering til Codex direkte i sessionen via eksisterende auth/config     | Hurtig uformel før-review → færre formelle kæde-runder spildt på tidlige fund; de formelle gates bliver i codex.sh-adapteren |
| **`/loop`** — kommando der gentager en prompt på interval i en åben session                                      | Poller og følger op automatisk uden manuel gentagelse                                                                | Overvågning af åbne Mathias-gates med ren rapportering — aldrig handling                                                     |
| **Statusline** — konfigurerbar statuslinje i terminalen, drevet af eget script                                   | Viser valgfri egen information konstant (scriptet modtager session-data som JSON)                                    | Kæde-tilstand (aktiv batch, KAEDE-STOP, afventende gates) altid synlig via `tilstand.mjs`-data                               |
| **Checkpointing** — automatiske snapshots af fil-ændringer ved hver prompt (`/rewind`)                           | Fortryder kode- og/eller samtale-tilstand uden manuel backup; 30 dages historik                                      | Fejlslagne ændringer rulles tilbage uden git-støj på kæde-branches                                                           |
| **`--from-pr`** — CLI-flag der genoptager den session, som skabte en given PR                                    | Henter den fulde kontekst fra PR-arbejdet tilbage med ét kald                                                        | Gate-/PR-opfølgning uden manuel genopbygning af kontekst                                                                     |
| **`/doctor` + `/context` + `/memory`** — indbyggede diagnose-kommandoer                                          | Viser hvad der FAKTISK er loadet: hooks, regler, CLAUDE.md, kontekst-forbrug                                         | Stille konfigurationsfejl fanges straks efter ændringer i hooks/regler                                                       |
| **Sandboxing** — OS-niveau-isolation af Bash-kommandoer (bubblewrap; virker på WSL2)                             | Begrænser filsystem- og netværksadgang for kommandoer uden ekstra prompts                                            | Ekstra sikkerhedslag oven på permissions + hooks i lange autonome kørsler                                                    |
| **Headless** — ikke-interaktiv kørsel (`claude -p`, `--bare`, JSON-schema-output)                                | Kører deterministisk og scriptbart: ingen auto-discovery, typed output                                               | Dirigent-kald med schema-valideret output i stedet for tekst-parsing                                                         |
| **Agent SDK** — programmerings-bibliotek (Python/TypeScript) til at bygge agenter på Claude Code-motoren         | Giver sessions/resume, hooks, structured outputs og cost-tracking som kode i stedet for CLI-wrapping                 | Kandidat til dirigent 2.0 ved næste større ombygning — ikke nu, kæden virker                                                 |
| **Agent view** — indbygget dashboard + supervisor-daemon for baggrundssessioner (`claude agents`)                | Dispatcher, overvåger og attacher parallelle sessioner; overlever terminal-luk                                       | Mulig forenkling af systemd-servicen — men governance-laget (gates, frys-SHA) dækkes ikke                                    |
| **Agent teams** — eksperimentel funktion: flere koordinerede sessioner med lead og delt task-liste               | Koordinerer multi-instans-arbejde native, med indbyrdes beskeder og uden eget orkestrerings-lag                      | Tættest på kæde-konceptet af alt nativt — men umoden (ingen resume); observér, brug ikke endnu                               |
| **Workflows** — script-styret orkestrering af subagenter i faser (`/workflows`, op til 1.000 agenter)            | Kører deterministisk fan-out: planen ligger i kode, ikke i modellens tur-for-tur-skøn; genoptagelig                  | Store batch-/audit-opgaver, f.eks. bred kode-recon                                                                           |
| **ultrareview** — cloud-baseret dybde-review med flåde af uafhængige reviewer-agenter (`/code-review ultra`)     | Verificerer (reproducerer) fund før rapportering → få falske positiver                                               | Supplement til Codex på store diffs — ikke gate-erstatning: Codex-uafhængighed er et governance-valg                         |
| **Routines** — cloud-agenter på Anthropic-infrastruktur, udløst af schedule, GitHub-events eller API             | Kører uafhængigt af egen maskine; frisk klon af repoet pr. kørsel                                                    | Natlige tjek selv når WSL er slukket — kræver GitHub-app-opsætning                                                           |
| **Worktrees** — isolerede git-checkouts pr. session (`claude --worktree`)                                        | Kører parallelle sessioner uden filkollision                                                                         | Sideløbende pakke-arbejde uden at forstyrre aktiv kæde-branch                                                                |
| **Auto mode** — permission-tilstand hvor en baggrunds-klassifikator vurderer handlinger i stedet for prompts     | Fjerner afbrydelser; tilliden afgrænses i prosa-regler (`autoMode.environment`)                                      | Lange kørsler uden prompt-stop — kræver omhyggelig tillids-afgrænsning før kæde-brug                                         |
| **Computer use** — desktop-app-funktion hvor Claude styrer apps via skærmen (Windows; research preview, Pro/Max) | Når til ting uden CLI/API: klikker, taster, ser skærmen (browsere view-only, terminaler click-only; off by default)  | Ingen kæde-anvendelse i dag. OBS: beskytter intet teknisk — gates bæres alene af Mathias' author-verificerede dokumenter     |
